# smart-lock-stm32

一个基于 STM32F103C8 的智能密码锁，能跑，能开锁，能改密码，断电密码还在。

---

## 这玩意儿能干啥

- **密码开锁**：待机按确认键，输6位密码，对了电机就转，凸轮顶开锁舌，5秒后自动关锁
- **掉电保存密码**：密码存在 Flash 最后一页，拔电池再插上还是原来的密码，不怕丢
- **随时改密码**：待机界面按返回键就能进配置模式，验证旧密码之后设新密码，输两次确认防手滑
- **各种提示音**：按键、开锁、错误、关锁、退格，蜂鸣器每种操作都有对应的叫声，不用瞎猜
- **OLED 显示**：中文/英文都能显示，状态一目了然
- **低功耗待机**：待机3分钟无操作自动进入睡眠，功耗从几十mA降到几十μA，按 `*` 键唤醒

## 项目文件结构

```
Hardware/          # 驱动层和业务模块
  Buzzer.c/h       # 蜂鸣器，各种提示音节奏
  Key.c/h          # 4×3矩阵键盘，行列扫描+状态机消抖
  Motor.c/h        # 直流电机控制
  OLED.c/h         # OLED 显示（江协科技 V2.0 软模拟 I2C）
  Password.c/h     # 密码模块，6位数字密码校验
  PWM.c/h          # TIM1_CH4 输出 10kHz PWM
  Store.c/h        # Flash 掉电存储，魔数+校验和双保险
  Switch.c/h       # 微动开关检测
  Unlock.c/h       # 开锁/关锁自动时序（凸轮逻辑）
  Config.c/h       # 配置模式，改密码的完整状态机

System/            # 系统级基础功能
  Delay.c/h        # 阻塞延时 + TIM4 非阻塞时基

User/              # 主程序和中断
  main.c           # 主循环，只做调度，逻辑很干净
  stm32f10x_conf.h # 标准库配置文件
  stm32f10x_it.c/h # 中断服务程序

Library/           # STM32F10x 标准库
Start/             # 启动文件和 CMSIS 核心文件
```

## 怎么用

### 1. 编译烧录

用 Keil µVision 5 打开 `Project.uvprojx`，编译下载到 STM32F103C8。

### 2. 默认密码

首次上电默认密码是 `123456`。密码存在 Flash 里，断电不会丢。

### 3. 开锁

- 待机界面显示 `Press CONFIRM`
- 按 `*`（确认键）进入密码输入，显示 `Password:`
- 输入6位数字，OLED 显示星号
- 按 `*`（确认键）验证
  - 正确：电机转动，凸轮顶开锁舌，微动开关到位后停转，显示 `OPEN`
  - 5秒后电机继续转动脱开，弹簧复位关锁，显示 `CLOSED`
  - 错误：蜂鸣器响两声，显示 `ERROR`，重新输入

### 4. 退格

密码输入时按 `#`（返回/退格键）：
- 有输入 → 删除最后一位（星号少一个）
- 无输入 → 返回待机界面

### 5. 低功耗待机与唤醒

- 待机界面显示操作提示，3分钟不按任何键，系统自动进入睡眠（OLED黑屏）
- 睡眠期间按任意数字键或 `#` 键 → **没反应**（系统醒来发现不是 `*`，继续睡）
- 按 `*` 键 → **唤醒**，显示 `密码:`，正常输入开锁

### 6. 修改密码

- 待机界面按 `#`（返回键），进入配置模式，显示 `Old Pwd:`
- 输入旧密码，按确认
  - 旧密码错误 → 显示 `ERROR`，直接退出
  - 旧密码正确 → 显示 `New Pwd:`
- 输入新密码（6位），按确认 → 显示 `Confirm:`
- 再输入一次新密码，按确认
  - 两次一致 → 写入 Flash，显示 `Saved`，新密码生效
  - 两次不一致 → 显示 `Mismatch`，修改失败
- 配置模式下**10秒无按键操作**自动退出回待机

## 一些技术细节（感兴趣可以看看）

### 为啥要把 main.c 拆成 Unlock/Config 模块？

原本一个 main.c 里塞了 400 多行，各种状态机全挤在一起，看一眼都得滚半天鼠标。后来拆了一下，main.c 只管调度和分发，开锁逻辑丢给 `Unlock.c`，改密码逻辑丢给 `Config.c`，各管一摊，清楚多了。

### Flash 存储怎么保证可靠？

最后一页 `0x0800FC00` 存了个 12 字节结构体：
- 4 字节魔数 `0xA5A5A5A5`：判断这页写过没有
- 6 字节密码
- 2 字节校验和：防止 Flash 偶尔抽风位翻转

上电先读 Flash，魔数和校验和都对才把密码加载到 RAM。不对的话说明是首次上电或者数据坏了，自动恢复默认密码 `123456`。

写 Flash 前先擦除整页（1KB），再按 32 位字写入，最后上锁。

### 按键输入：从独立按键到矩阵键盘

最早是12路独立按键，每个键各接一根GPIO，占引脚太多。后来换成了 **4×3 矩阵键盘**，4根行线（PA0~PA3）做推挽输出，3根列线（PA4~PA6）做上拉输入，7根线搞定12个键。

扫描逻辑：逐行置低电平，读列线，行列交叉点就是按下的键。扫描后加 10us 延时等电平稳定，不然容易误判。

`*` 映射为确认键，`#` 映射为返回/退格键，`0~9` 还是数字。

消抖还是原来的20ms状态机，只是"读电平"变成了"矩阵扫描推断电平"，核心逻辑一点没变。

**踩坑**：列线接反了，导致2和3、5和6、8和9、0和#整列对调。调试时临时把星号显示改成数字显示，按一圈就知道哪个键对应什么值，定位问题后对调两根杜邦线解决。

### 低功耗与Stop模式

待机3分钟无操作，系统进入 **Stop模式**。CPU和所有时钟全停，但SRAM里的数据还在（密码不丢）。只有按 `*` 键才能正常唤醒，其他键按了系统醒来扫一眼发现不是 `*`，翻个身继续睡。

列线（PA4~PA6）配置为EXTI下降沿中断源，任意键按下都会触发中断唤醒CPU。唤醒后 `SystemInit()` 重新配置72MHz时钟，然后矩阵扫描做软件过滤。

为什么不选Sleep（降太少）或Standby（丢数据）？Stop是功耗和功能的sweet spot：几十μA + 数据保留 + 按键唤醒。

### 电机控制

凸轮结构，单向旋转就行，不需要正反转。PWM 10kHz，占空比 80%，力气够，噪音小。
