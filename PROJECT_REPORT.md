# 基于 STM32F103 的智能电子密码锁系统设计与实现

## 摘要

本文设计并实现了一套基于 STM32F103C8T6 微控制器的智能电子密码锁系统。系统采用直流减速电机配合凸轮机构实现锁舌的驱动，通过微动开关进行到位检测，利用 OLED 显示屏提供人机交互界面，并采用蜂鸣器实现多维度的音频反馈。针对传统密码锁断电丢失密码的缺陷，本系统设计了基于 Flash 存储的掉电数据保护机制，引入魔数校验与累加和校验双重验证策略，确保存储可靠性。此外，系统实现了配置模式，支持用户在不借助外部工具的情况下完成密码的安全修改，并引入 STOP 低功耗模式，在待机状态下将系统功耗降低至微安级别，延长电池续航时间。软件架构采用模块化设计思想，将业务逻辑按功能域拆分，降低了系统耦合度，提升了代码的可维护性。

**关键词**：STM32；电子密码锁；Flash 存储；状态机；嵌入式系统

## 1. 项目概述

### 1.1 研究背景与意义

随着物联网技术与嵌入式系统的快速发展，智能门禁系统在家庭、办公及仓储场景中得到了广泛应用。相较于传统的机械锁具，电子密码锁具有无需携带物理钥匙、密码可动态变更、易于集成远程管理等优势。然而，市面上部分低端电子锁产品存在密码断电易失、交互体验差、机构可靠性不足等问题。

本课题旨在设计一款成本可控、功能完善、交互友好的智能密码锁原型系统，重点解决以下技术问题：
- 密码及配置信息的非易失性存储
- 电机驱动机构的精准时序控制与到位检测
- 人机交互界面的实时响应与防闪烁显示
- 按键输入的可靠消抖与事件分发

### 1.2 系统总体指标

| 指标项 | 参数 |
|--------|------|
| 主控芯片 | STM32F103C8T6（ARM Cortex-M3，72MHz） |
| 密码长度 | 6 位十进制数字 |
| 存储介质 | 片内 Flash（末页 1KB） |
| 电机类型 | 直流减速电机 + 凸轮机构 |
| PWM 频率 | 10 kHz |
| 显示设备 | 0.96 英寸 OLED（128×64，I2C 接口） |
| 输入设备 | 4×3 矩阵键盘（薄膜按键） |
| 待机功耗 | 约 30~50 μA（STOP 模式） |
| 供电电压 | 5V / 3.3V |

## 2. 系统硬件设计

### 2.1 主控单元

系统核心采用 STM32F103C8T6 微控制器，其内部资源分配如下：
- **TIM1**：通道 4（PA11）输出 PWM 信号，用于电机调速
- **TIM4**：提供 1ms 时基中断，为系统提供非阻塞延时能力
- **GPIO**：PA0~PA6 用于矩阵键盘行列扫描，PB8/PB9 用于 OLED 软模拟 I2C，PB12/PB13/PA11 用于电机驱动，PA12 用于微动开关检测
- **Flash**：利用 64KB Flash 的末页（地址 0x0800FC00）作为非易失性数据存储区

### 2.2 电机驱动机构

锁具执行机构采用直流减速电机配合凸轮的单向旋转方案：
- 电机正转时，凸轮顶开锁舌，压缩弹簧
- 凸轮顶点触发微动开关，系统检测到到位信号后停止电机，锁舌保持开启状态
- 延时 5 秒后，电机继续同向旋转，凸轮脱离锁舌，弹簧弹力驱动锁舌自动复位
- 该方案无需电机反转，简化了驱动电路与控制逻辑

电机驱动采用半桥驱动模块，通过 PB12/PB13 控制转动方向，PA11 输出 PWM 调速信号，占空比设定为 80%。

### 2.3 人机交互单元

**显示模块**：采用基于 SSD1306 控制器的 0.96 英寸 OLED 显示屏，通过 PB8（SCL）、PB9（SDA）软件模拟 I2C 通信。显示策略采用"状态切换时全局刷新，状态内局部刷新"的机制，有效消除了星号输入过程中的画面闪烁问题。

**音频反馈模块**：PB10 驱动有源蜂鸣器，系统定义了五种提示音模式：
| 事件类型 | 节奏特征 |
|----------|----------|
| 按键按下 | 单短音（约 30ms） |
| 开锁成功 | 短-长-短 三声 |
| 密码错误 | 双短音 |
| 关锁完成 | 短-短-长 三声 |
| 退格操作 | 中音（约 100ms） |

**输入模块**：采用 4×3 矩阵键盘设计，覆盖数字 0-9、确认键（*）及返回键（#）。行线（4 根）配置为推挽输出，列线（3 根）配置为上拉输入，通过逐行扫描方式检测按键。扫描策略为：逐行置低电平，读取列线状态，行列交叉点即为按下的键。按键检测同样采用基于时间戳的状态机消抖算法，消抖阈值 20ms，支持事件缓冲。

### 2.4 引脚分配表

| 功能模块 | 信号名称 | 引脚编号 | 工作模式 |
|----------|----------|----------|----------|
| OLED 显示 | SCL | PB8 | 开漏输出（软 I2C） |
| OLED 显示 | SDA | PB9 | 开漏输出（软 I2C） |
| 电机驱动 | IN1 | PB12 | 推挽输出 |
| 电机驱动 | IN2 | PB13 | 推挽输出 |
| 电机驱动 | PWM | PA11 | 复用推挽（TIM1_CH4） |
| 矩阵键盘 | 行线 R0~R3 | PA0~PA3 | 推挽输出（默认高电平） |
| 矩阵键盘 | 列线 C0~C2 | PA4~PA6 | 上拉输入 |
| 蜂鸣器 | BEEP | PB10 | 推挽输出 |
| 位置检测 | SWITCH | PA12 | 上拉输入 |

## 3. 系统软件设计

### 3.1 软件架构

系统软件采用分层架构设计：

- **硬件抽象层（HAL）**：基于 STM32 标准外设库，提供寄存器级的底层操作
- **驱动层（Driver）**：封装各外设初始化及基础控制接口（OLED、PWM、GPIO 等）
- **业务逻辑层（Service）**：实现密码管理、存储管理、开锁时序、配置流程等核心业务
- **应用调度层（App）**：主循环负责事件监听与状态分发

模块划分遵循"高内聚、低耦合"原则，各模块头文件对外暴露最小必要接口。主程序文件 `main.c` 仅承担系统初始化和事件调度职责，具体的业务逻辑由 `Unlock.c`、`Config.c`、`Password.c`、`Store.c` 等模块独立实现。

### 3.2 主状态机设计

系统运行期由主状态机统一调度，状态定义如下：

```
STATE_IDLE            -- 待机状态，等待用户触发
STATE_PASSWORD_INPUT  -- 密码输入状态，接收数字录入
STATE_UNLOCKING       -- 开锁执行状态，电机转动并检测到位
STATE_UNLOCKED        -- 开锁保持状态，等待自动关锁延时
STATE_LOCKING         -- 关锁执行状态，电机继续转动复位
STATE_CONFIG_VERIFY   -- 配置模式：验证旧密码
STATE_CONFIG_SET      -- 配置模式：输入新密码
STATE_CONFIG_CONFIRM  -- 配置模式：确认新密码
```

其中开锁相关的三个状态（UNLOCKING / UNLOCKED / LOCKING）已封装至 `Unlock` 模块，由该模块内部维护子状态机，主循环通过 `Unlock_Tick()` 周期性驱动。配置相关的三个状态封装至 `Config` 模块，通过 `Config_HandleEvent()` 处理按键事件。

### 3.3 密码管理模块

密码管理模块（`Password.c`）提供以下核心能力：
- 输入缓存管理：支持逐位录入，满 6 位后自动拒绝超限输入
- 密码校验：逐位数组比对，支持固定时间比较（可扩展防侧信道攻击）
- 退格操作：带下溢保护，空输入时退格无效
- 密码设置接口：供 Flash 加载及配置模式调用

当前有效密码存储于 RAM 中的 `s_password[6]` 数组，上电时由存储模块从 Flash 加载。

### 3.4 Flash 存储模块

为实现密码的掉电保持，系统在 Flash 末页（0x0800FC00）存储配置数据，数据结构定义如下：

```c
typedef struct {
    uint32_t magic;      /* 魔数 0xA5A5A5A5，用于识别已初始化 */
    uint8_t  pwd[6];     /* 6 位密码 */
    uint16_t checksum;   /* 前 10 字节的累加和校验 */
} LockConfig_t;
```

存储操作遵循 Flash 物理特性：
1. 写入前必须先执行页擦除（1KB 整页）
2. 写入粒度为 32 位字（Word），12 字节数据分 3 个 Word 顺序写入
3. 操作前解锁 Flash 控制寄存器，操作后立即上锁
4. 读取时进行魔数校验与校验和双重验证，任一失败则回退至默认密码

### 3.5 配置模式设计

系统支持用户直接在设备上修改密码，无需借助外部烧录工具。配置流程设计如下：

1. **触发条件**：待机界面下按下返回键
2. **旧密码验证**：用户输入当前密码，验证失败直接退出并提示错误
3. **新密码录入**：验证通过后进入新密码输入，要求必须输满 6 位
4. **二次确认**：再次输入新密码，系统逐位比对两次录入结果
5. **持久化存储**：比对一致后调用 `Storage_WritePassword()` 写入 Flash

**安全策略**：
- 配置全程启用 10 秒超时机制，最后一次按键后 10 秒内无操作自动退出
- 旧密码验证错误不返回配置入口，直接退回待机，防止暴力尝试
- 两次输入不一致时拒绝写入，避免误触导致密码被篡改

### 3.6 矩阵键盘扫描与按键消抖

#### 3.6.1 矩阵键盘扫描机制

系统采用 4×3 矩阵键盘替代传统的独立按键方案，仅需 7 个 GPIO 即可实现 12 个按键的输入。扫描逻辑如下：

1. 将所有行线置为高电平
2. 将当前扫描行置为低电平
3. 延时 10μs 等待电平稳定
4. 读取 3 根列线的电平状态
5. 若某列线为低电平，则当前行与该列交叉处的按键被按下
6. 遍历 4 行完成整盘扫描

扫描函数返回按键索引（0~11）或无效值（-1）。

#### 3.6.2 按键消抖与事件分发

在矩阵扫描基础上，系统为每个物理按键（共 12 个）维护独立的消抖状态机：

```c
typedef enum {
    KEY_STATE_IDLE,
    KEY_STATE_PRESS_DEBOUNCE,
    KEY_STATE_PRESSED,
    KEY_STATE_RELEASE_DEBOUNCE
} KeyState_t;
```

每轮扫描后，将当前扫描结果映射为各按键的"虚拟电平"（扫描命中则为低电平，否则为高电平），喂入对应的状态机进行 20ms 消抖判定。事件缓冲机制确保单次按键仅产生一个有效事件，避免重复触发。

按键索引到事件码的映射关系如下：

| 索引 | 物理按键 | 事件码 |
|------|----------|--------|
| 0~8 | 1~9 | 0x01~0x09 |
| 9 | * | KEY_CONFIRM（0xFE） |
| 10 | 0 | 0x00 |
| 11 | # | KEY_BACK（0xFD） |

### 3.7 低功耗管理

为延长电池供电场景下的续航时间，系统实现了基于 STM32 STOP 模式的低功耗待机策略。

#### 3.7.1 STOP 模式工作原理

STOP 模式下，Cortex-M3 内核停止运行，HSE、HSI 及 PLL 等时钟源全部关闭，但电压调节器保持开启，SRAM 与寄存器内容完整保留。系统功耗从正常工作时的约 15~25mA 降至约 30~50μA，降低了三个数量级。唤醒后通过 `SystemInit()` 重新配置时钟树，恢复 72MHz 主频。

#### 3.7.2 唤醒源设计

系统将矩阵键盘的 3 根列线（PA4~PA6）配置为 EXTI 外部中断源，下降沿触发。任意按键按下时，对应列线被拉低，产生 EXTI 中断事件唤醒 CPU。 EXTI 线映射关系如下：

| EXTI 线 | GPIO 引脚 | 列线 | NVIC 中断向量 |
|---------|-----------|------|--------------|
| EXTI4 | PA4 | C0 | EXTI4_IRQn（独立） |
| EXTI5 | PA5 | C1 | EXTI9_5_IRQn（共享） |
| EXTI6 | PA6 | C2 | EXTI9_5_IRQn（共享） |

#### 3.7.3 软件过滤机制

由于矩阵键盘的行列共享特性，硬件层面无法单独识别 `*` 键的按下事件（同一列的 `1`、`4`、`7` 也会触发 EXTI）。系统采用**软件过滤**策略：

1. 任意键按下 → EXTI 唤醒 CPU
2. 唤醒后执行 `SystemInit()` 恢复时钟，延时 20ms 等待时钟稳定与按键消抖
3. 执行矩阵扫描，获取唤醒按键的事件码
4. 若为 `*` 键（`KEY_CONFIRM`），退出内层循环，正常进入密码输入流程
5. 若为非 `*` 键，重新进入 STOP 模式，用户无感知

该机制以不到 1ms 的额外唤醒-判断-再睡眠开销，实现了"仅 `*` 键有效唤醒"的用户体验。

#### 3.7.4 进入 STOP 的时机与超时策略

系统仅在 `STATE_IDLE`（待机界面）状态下判断进入 STOP 模式，确保电机运行、密码输入、配置流程等关键阶段不会被打断。超时阈值设定为 3 分钟（180 秒），用户在待机界面无操作 3 分钟后系统自动进入睡眠。每次检测到有效按键操作时，超时计时器重置。

```c
if (GetTick() - g_idleTick > IDLE_TIMEOUT_MS)  /* 180000ms = 3min */
{
    PWR_EnterSTOPMode(PWR_Regulator_ON, PWR_STOPEntry_WFI);
    /* 唤醒后恢复时钟并过滤按键 */
}
```

### 4.2 性能测试

- **系统响应 latency**：按键到蜂鸣器反馈 < 5ms，到 OLED 刷新 < 20ms
- **开锁总时序**：电机启动 → 到位停止（约 200-500ms，取决于机械结构）→ 保持 5s → 关锁 500ms
- **Flash 写入耗时**：页擦除 + 3 Word 写入，总耗时约 20-40ms
- **功耗估算**：
  - 正常工作（MCU + OLED）：约 15-25mA
  - STOP 低功耗模式：约 30~50μA
  - 电机运行时：约 150-300mA
  - 以 2000mAh 电池估算，STOP 模式下理论待机时间超过 4 年

## 5. 总结与展望

### 5.1 工作总结

本系统完成了以下设计目标：
1. 实现了基于 STM32F103 的完整密码锁功能，包含输入、校验、电机驱动及自动关锁时序
2. 设计了可靠的 Flash 掉电存储方案，通过魔数与校验和双重机制保证数据完整性
3. 实现了安全的配置模式，支持用户现场修改密码，具备超时保护与二次确认机制
4. 采用 STOP 低功耗模式实现待机节能，功耗降低三个数量级，延长电池续航
5. 采用模块化软件架构，将业务逻辑按功能域拆分，提升了系统的可维护性与可扩展性

### 5.2 不足与改进方向

1. **安全性增强**：当前密码比对为逐位数组比较，后续可引入恒定时间算法（Constant-Time Comparison）抵抗侧信道攻击
2. **多模态开锁**：可扩展指纹模块（如 AS608）、RFID 读卡器或蓝牙模块，实现多因素认证
3. **远程管理**：集成 ESP8266 等 WiFi 模组，支持密码远程下发与开锁记录上传
4. **异常处理**：当前 Flash 写入失败仅返回错误码，未做重试与降级处理，后续可完善容错机制
5. **功耗进一步优化**：当前 STOP 模式下电压调节器保持开启，后续可尝试低功耗电压调节器模式，或引入 RTC 定时唤醒实现间歇性键盘扫描
