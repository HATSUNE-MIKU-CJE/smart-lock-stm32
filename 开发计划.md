# STM32智能密码锁 — 软件开发计划（精简版）

> 当前聚焦：软件实现层  
> 硬件已确定：STM32F103C8 + 12独立按键 + OLED(PB6/7/8/9) + 舵机(PA11/TIM1) + 蜂鸣器(PB10) + AS608指纹(USART1) + BY9301语音(USART3) + 端子检测

---

## 一、软件架构总览（三层）

```
┌─────────────────────────────────────┐
│  应用层    状态机 + 菜单              │  ← 决定"现在该干嘛"
├─────────────────────────────────────┤
│  业务层    密码管理 + 配置存储         │  ← 决定"这个密码对不对"
├─────────────────────────────────────┤
│  驱动层    Key / OLED / Servo / Buzzer │  ← 决定"让硬件怎么动"
│           FingerPrint / Voice        │
└─────────────────────────────────────┘
```

**原则**：下层不依赖上层。驱动层只汇报事件，不判断业务逻辑。

## 二、功能 ↔ 技术栈 ↔ 元件 对照表

| 用户功能 | 软件技术点 | 硬件元件/接口 | 新人需知 |
|---------|-----------|-------------|---------|
| 密码输入 | 独立按键扫描 + 输入缓存数组 `buf[6]` + 长度计数器 | 12个独立按键（IPU接GND）| 消抖用状态机，不要Delay堵主循环 |
| 密码验证 | `uint32_t` 整数比较（如 `input == 123456`）| 纯软件 | 数组攒够6位后转成整数再比，别逐位比 |
| 掉电保存 | Flash页擦除 + 结构体写入 + 魔数校验 | STM32内部Flash（末页 `0x0800FC00`）| **必须先擦除再写入**，Flash只能把1变0 |
| 舵机开锁 | TIM1_PWM 50Hz，CCR 500~2500 | SG90（信号PA11，电源外接5V）| 舵机电源不可从STM32取电，会烧 |
| 到位检测 | GPIO输入 + 电平轮询/EXTI中断 | 微动开关/端子 | 比延时更可靠，机械阻力变化也不怕 |
| 自动关锁 | 软件定时计数（SysTick或简单累加）| 纯软件 | 在状态机里做超时跳转，别用Delay阻塞 |
| 指纹解锁 | UART串口收发 + AS608指令包组帧 | AS608（USART1 PA9/10）| TX/RX**交叉接**，波特率57600 |
| 语音播报 | UART发指令控制播放 | BY9301（USART3 PB10/11）| TX/RX**交叉接**，波特率9600 |
| OLED显示 | I2C软模拟 + 帧缓冲刷新 | SSD1306（PB6/7 SCL/SDA）| 中文显示需确保字模已取好 |
| 蜂鸣器提示 | GPIO翻转 + 软件定时鸣响 | 有源蜂鸣器（PB10）| 高电平响还是低电平响，取决于模块 |

## 三、分阶段开发计划（只做5步）

### 阶段1：核心骨架 —— 固定密码能开锁
**目标**：上电→按确认键→输入6位密码→按确认→舵机转动→端子到位→蜂鸣器响。

**涉及模块**：Key（12按键）、OLED、Servo、Buzzer、端子GPIO、简易状态机

**技术细节**：
- **状态机就3个状态**：`STATE_IDLE`（待机）→ `STATE_INPUT`（输入中）→ `STATE_UNLOCKED`（已开锁）
- **输入缓存**：`uint8_t buf[6]` + `uint8_t len`，满6位后按确认才验证
- **事件缓冲**：Key模块只产出事件（如 `EVT_NUM_1`、`EVT_CONFIRM`），状态机消费事件
- **消抖**：每个按键独立状态机（IDLE → PRESS_DEBOUNCE → PRESSED → RELEASE_DEBOUNCE）
- **端子检测**：舵机转动后循环读GPIO，读到有效电平即停止转动

**新人坑点**：
- 按键必须等**释放后才能识别下一次**，否则按住不放会疯狂触发
- 状态机用 `switch(g_state)` 套 `switch(event)`，不要写成一堆if嵌套
- 先不要碰Flash、不要碰低功耗，密码直接写死在代码里

---

### 阶段2：数据持久化 —— 断电密码不丢
**目标**：密码存在Flash里，首次上电自动写入默认密码（如 `123456`），支持以后修改。

**涉及模块**：Storage（新建）

**技术细节**：
- **存储位置**：STM32F103C8 最后一页 `0x0800FC00`（1KB）
- **数据结构**：
  ```c
  typedef struct {
      uint32_t magic;      // 魔数 0xA5A5A5A5，判断"是不是第一次上电"
      uint32_t user_pwd;   // 用户密码
      uint32_t admin_pwd;  // 管理员密码（用于重置）
      uint16_t checksum;   // 校验和
  } LockConfig;
  ```
- **写入流程**：解锁Flash → 擦除整页（全变0xFF）→ 写入结构体 → 上锁
- **读取流程**：上电时读Flash地址，如果 `magic != 0xA5A5A5A5`，说明是首次上电，写入默认密码

**新人坑点**：
- **Flash写入前必须先擦除**，擦除后整页全是0xFF
- Flash只能把 **1改成0**，不能把0改成1（所以必须擦除后再写）
- 不要频繁写Flash（有寿命限制，约1万次），改密码时才写，平时只读

---

### 阶段3：端子闭环 + 自动关锁
**目标**：开锁时舵机转到端子触发位自动停；5秒后自动关锁。

**涉及模块**：状态机 + Servo + 端子GPIO

**技术细节**：
- **闭环流程**：`STATE_UNLOCKED` → 舵机慢速转动 → 循环检测端子电平 → 到位停止 → 启动5秒倒计时
- **超时机制**：在状态机里记录进入 `STATE_UNLOCKED` 的时间戳，每轮while(1)检查 `当前时间 - 进入时间 > 5秒`
- **关锁流程**：倒计时到 → 舵机反向转动 → 可选检测端子释放（或转到0°）→ 回到 `STATE_IDLE`

**新人坑点**：
- 不要用 `Delay_s(5)` 阻塞主循环来做超时，否则这5秒内按键完全失效
- 用**时间差判断**：记录一个 `uint32_t unlock_tick`，主循环里 `if (GetTick() - unlock_tick > 5000)`
- 端子如果接的是机械开关，注意**抖动**，首次检测到后建议延迟10ms再确认

---

### 阶段4：指纹 + 语音
**目标**：录入指纹、指纹验证开锁、关键节点语音播报。

**涉及模块**：FingerPrint（AS608）、Voice（BY9301）

**技术细节**：
- **AS608指纹流程**：
  - 录入：`GetImage` → `GenChar(1)` → 再按一次 → `GenChar(2)` → `Match`比对 → `RegModel`合并 → `StoreChar(id)`
  - 验证：`GetImage` → `GenChar(1)` → `Search`搜索 → 返回ID和分数（分数>100认为通过）
  - 通信：USART1，波特率57600，数据包格式为 `包头+地址+包标识+长度+内容+校验和`
- **BY9301语音流程**：
  - USART3发送播放指令（波特率9600），如播放指定编号音频
  - 触发时机：密码正确/错误、指纹匹配成功、进入报警等事件发生时调用

**新人坑点**：
- AS608和BY9301都是**UART模块，TX接RX，RX接TX**，别接反
- AS608的校验和容易算错，建议找份现成的组包函数移植，不要手算
- 语音模块播报需要时间（几百毫秒），不要连续发多条指令，等播完再发下一条

---

### 阶段5：菜单整合 + 报警 + 完善
**目标**：完整产品体验。密码修改、指纹管理、错误锁定、管理员模式。

**涉及模块**：Menu（菜单树）、StateMachine（完整版）

**技术细节**：
- **菜单结构**：单层菜单即可，不用多级嵌套（降低复杂度）
  - 选项：1.修改密码 2.录入指纹 3.删除指纹 4.返回
  - 用数字键选择，`#`确认，`*`返回
- **密码修改流程**：输入旧密码 → 验证通过 → 输入新密码 → 再输入一次确认 → 写入Flash
- **错误锁定**：连续3次密码错误或指纹失败 → 进入 `STATE_ALARM` → 蜂鸣器长响30秒 → 期间拒绝一切输入
- **管理员模式**：长按某组合键（或输入管理员密码）→ 进入高级菜单，可绕过锁定

**新人坑点**：
- 菜单也是状态机的一部分，建议用 `STATE_MENU_MAIN`、`STATE_MENU_SET_PWD` 等状态，不要另起一套逻辑
- 修改密码时，**新密码需要输入两次确认**，防止按错
- 报警状态下也要检测按键（用于管理员解锁），不要完全屏蔽输入
